# Recipe Tin UI

Recipe Tin UI is the component library and design system for building consistent, accessible, and scalable recipe application experiences. It provides reusable components, design foundations, and guidelines for teams designing and developing interfaces across the platform.

Recipe Tin UI helps teams move faster, collaborate more effectively, and deliver cohesive experiences for home cooks and food enthusiasts. By aligning design and engineering around shared standards, every interface we build reflects the brand, supports usability, and earns the trust of users who rely on the platform to discover, save, and cook great food.

By using the system across all digital touchpoints, our products:

- Look consistent and cohesive
- Behave predictably and intuitively
- Meet accessibility standards
- Scale efficiently as the platform evolves

## How It Works

The most tangible outputs of Recipe Tin UI are reusable components — buttons, inputs, recipe cards, navigation, ratings, progress indicators, and more.

These components:

- Follow Recipe Tin's brand foundations (color, typography, spacing, iconography)
- Adapt responsively across screen sizes and devices
- Are built with accessibility in mind
- Support real recipe and nutrition workflows

Each component is documented with usage guidance, states, accessibility notes, and implementation details to ensure consistency from design to production.

## Usage

### Using Components

Import components directly:

```tsx
import { Button } from "recipe-tin-ui";

export function Example() {
  return <Button variant="primary">Browse Recipes</Button>;
}
```

### Using Design Tokens

Design tokens are available for direct use in CSS.

```css
.my-card {
  background: var(--color-primary-800);
  border-radius: var(--border-radius-md);
}
```

---

### Design Foundations

The system is built on a clear set of foundations:

- **Design Tokens** – Colors, spacing, typography scales, and border radius values defined as CSS custom properties
- **Color System** – Brand colors (primary green, secondary teal, tertiary indigo), neutral grays, and semantic states (success, warning, error)
- **Typography** – Heading and text scales using Roboto, with consistent weights and sizing
- **Accessibility Standards** – Contrast ratios, keyboard support, focus states, and screen reader considerations

These foundations ensure flexibility while maintaining consistency across teams and surfaces.

---

### React Component Library

On this site you'll find the full React component library and documentation.

Developers can:

- Install components via npm
- Reference live examples and usage guidelines
- Follow accessibility and implementation best practices
- Leverage design tokens directly in code

Components are organized into four categories:

- **Form** – Button, ButtonIcon, FormInput, FormSearch
- **Layout** – Stack, Inline, Page, NavBar, Footer
- **Display** – Heading, Image, Ratings, ReadMore, RecipeCard, RecipeGallery, RecipeDuration, IngredientsItem
- **Data Visualization** – ProgressBar, CaloriesBar, NutrientBar

Each component is built to be composable, scalable, and aligned with real product workflows.

---

### CSS & Design Tokens

The design system exposes all foundations as CSS custom properties.

Teams may use:

- Component classes
- Design tokens directly in CSS or inline styles
- Utility-based approaches

This ensures the system can support multiple product surfaces and technical stacks while preserving visual and behavioral consistency.

## Why It Matters

Recipe Tin serves users who want a reliable, delightful experience when discovering and cooking recipes. Clarity, ease of use, and visual appeal are essential.

The design system helps us:

- Reduce design and engineering redundancy
- Improve onboarding for new team members
- Deliver features faster
- Maintain brand integrity
- Create predictable, confidence-inspiring experiences

Consistency in UI leads to confidence in the product.
