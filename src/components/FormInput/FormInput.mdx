# Form Input

A controlled text input with an associated label, optional icon, error message, and support for all standard HTML input types.

## Guidelines

- Always provide an `id` and a `label` — the label is linked to the input for accessibility.
- Use `hideLabel` only when an icon or surrounding context makes the purpose of the input self-evident (e.g. a search field).
- Use the `error` prop to display inline validation messages. Always pair errors with a visible message, not colour alone.
- Mark fields as `required` when they must be completed before the form can be submitted.

### Prop reference

| Prop | Type | Default | Description |
|---|---|---|---|
| `id` | `string` | — | Links the `<label>` to the `<input>` via `htmlFor` |
| `label` | `string` | — | Visible label text for the input |
| `value` | `string` | — | Controlled value of the input |
| `onChange` | `(e: ChangeEvent) => void` | — | Change handler for the controlled input |
| `type` | `"text" \| "email" \| "password" \| "number" \| "search" \| "tel" \| "url"` | `"text"` | HTML input type |
| `placeholder` | `string` | `undefined` | Placeholder text shown when the input is empty |
| `icon` | `ReactElement` | `undefined` | Optional icon rendered inside the input at size 18 |
| `error` | `string` | `undefined` | Inline error message displayed below the input |
| `required` | `boolean` | `false` | Marks the field as required and appends an asterisk to the label |
| `disabled` | `boolean` | `false` | Prevents interaction with the input |
| `hideLabel` | `boolean` | `false` | Visually hides the label while keeping it accessible |
