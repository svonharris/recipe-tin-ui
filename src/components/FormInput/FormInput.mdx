# Form Input

A controlled text input with an associated label, optional icon, error message, and support for all standard HTML input types.

## Guidelines

- Always provide an `id` and a `label` — the label is linked to the input for accessibility.
- Use `hideLabel` only when an icon or surrounding context makes the purpose of the input self-evident (e.g. a search field).
- Use the `error` prop to display inline validation messages. Always pair errors with a visible message, not colour alone.
- Mark fields as `required` when they must be completed before the form can be submitted.
